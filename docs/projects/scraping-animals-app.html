Scraping Animals Pipeline & Search Engine

A full-stack data engineering project designed to scrape animal profiles from A-Z Animals, store them in MongoDB, index them in Elasticsearch, and expose them via an interactive Streamlit Web App.

Overview & Collaboration

This repository represents an end-to-end data pipeline built entirely in pair-programming with an AI assistant. It demonstrates expertise in handling extraction logic, database management, search index creation, and interactive data visualization all within a containerized Docker architecture.

Core Features

  • Robust Scraper (Scrapy): Employs generic dynamic extraction for physical characteristics and "fun facts". Safely bypasses WAF/403 protections using scrapy-impersonate, feeding data directly into a MongoDB pipeline.
  • Search Engine (Elasticsearch): Enables full-text search with fault tolerance (fuzzy search). Includes an automatic fallback to a simple local search if Elasticsearch is temporarily unavailable.
  • Web App (Streamlit): Provides an interface to interactively explore the data with filters customized by diet, habitat, and country. Includes real-time statistics and Folium cartography.

System Architecture

The pipeline guarantees a seamless flow of data across multiple decoupled components:

  • Extraction: Scrapy extracts JSON payloads from A-Z Animals.
  • Storage: Data is persisted in MongoDB via PyMongo pipelines.
  • Indexing: Data is synchronized from Mongo to Elasticsearch for high-performance querying capabilities. Kibana is available for underlying index management.
  • Interaction: The Streamlit frontend acts as the user gateway, utilizing a custom SearchClient to interface securely with Elasticsearch and Mongo.

Deployment & Structure

The entire stack is containerized locally using Docker Compose allowing 1-click execution. The repository is clearly divided into specific modules:

  • scrapy/: Contains the Spiders, pipelines, and settings.
  • webapp/: The Streamlit application source.
  • scripts/: Utility scripts (e.g., triggering ES indexing).
  • tests/: A suite for unit and integration testing via pytest and flake8.

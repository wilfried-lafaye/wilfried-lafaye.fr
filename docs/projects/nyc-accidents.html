NYC Accidents SQL Model & Analysis

A data engineering and analysis project focused on understanding database normalization, SQL creation, and data modeling using real-world NYC traffic accident data.

Overview

This project transforms raw, flat CSV data from NYC Open Data into a structured, relational SQLite database. It serves as a practical demonstration of moving data from an unoptimized state into a normalized architecture (targeting 3rd Normal Form).

Learning Objectives

  • Database Normalization: Structuring flat data efficiently to eliminate redundancy and ensure data integrity.
  • SQL Mastery: Writing complex SQL queries to extract meaningful insights using joins, aggregations, and advanced filtering.
  • Application Development: Bridging a robust SQL backend with a modern interactive frontend using Python and Streamlit.

Features & Architecture

  • Data Pipeline: Fetches live data via the NYC Open Data API, cleans the raw inputs, and automatically splits it into relational tables.
  • Interactive Dashboards: Visualizes accidents by borough, time of day, and severity using Plotly Express.
  • Geospatial Mapping: Plots specific accident coordinates on interactive maps.
  • SQL Playground: Provides a built-in interface to write and execute custom SQL queries directly against the new SQLite database.

Database Schema

The normalization process yields five core relational tables:

  • Lieu: Stores unique location properties (Latitude, Longitude, Borough, Street names).
  • Accident: Captures the core accident events linked functionally to Locations.
  • VehiculeType: A dedicated lookup table for various vehicle types (e.g., Sedan, SUV, Bike).
  • VehiculeInAccident: A join table tracking the specific vehicles involved in respective crashes.
  • FacteurContributif: Catalogs the variables contributing to the accidents, such as driver distraction.
